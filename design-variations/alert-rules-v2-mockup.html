Design Changes from v1
Removed: type field All rules are "simple" — correlation/baseline have no differentiated UI yet. Removed from form, stored as default.
Removed: per-condition severity Rule-level severity is what matters for notifications. Per-condition severity auto-set to match rule. Simplifies the condition row from 5 columns to 4.
Added: Active toggle on create Users can create rules in inactive state for testing before enabling.
Added: AND label between conditions Explicitly shows that all conditions must be met simultaneously. No ambiguity.
Added: Live preview sidebar Human-readable summary of what the rule will do. Updates as you type. Shows conditions, scope, severity, cooldown in plain language.
Added: "How it works" box Explains AND logic, duration=0, and cooldown. Reduces support questions.
Duration label: "For at least" Changed from "Duration (min)" to "For at least X min" — reads as a sentence: "temperature is above 8 for at least 5 min".
Show: Delete button added Delete was only on index. Now available on show page too (with confirmation).
