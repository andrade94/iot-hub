📐 v2 Changes from v1 Mockup

  • Zoom controls added to toolbar: −/+/percentage. Scroll wheel zooms. Canvas area scrolls for pan.
  • Contextual right panel: Device selected → shows device properties (name, model, DevEUI, zone dropdown, status, reading, battery, position, "View Details" link). Zone selected → shows zone properties (name, color, devices list, boundary coords, delete).
  • Search in Devices sidebar with search input at the top.
  • Tool selector replaces vague "Edit Zones" button: [Select] [Draw Zone]. Draw Zone mode shows instruction text and Cancel button.
  • Zone colors on canvas: Each zone rectangle is tinted with its assigned color (green for coolers, amber for kitchen, blue for freezer, gray for utility).
  • Resize handles on selected zone rectangle (4 corners: nw/ne/sw/se).
  • Legend on canvas bottom-left: Online (green), Warning (amber), Offline (red).
  • Undo button in header (ghost style) + keyboard shortcuts in status bar (Del, Esc, scroll to zoom).
  • Empty state (State 2): Full-height CTA with drag-and-drop upload zone when no floor plan exists.
  • Properties panel close button (✕) to collapse the panel for more canvas space.
  • Device "View Details" link in properties → navigates to /devices/{id}.
  • Right panel sidebar is 260px (down from 280px) and left sidebar is 300px (down from 320px) — gives more canvas space on laptops.

✅ Implementation Plan (7 phases)

  • Phase 1: Route + page shell: GET /sites/{site}/layout → sites/layout.tsx. Three-panel layout with sidebar tabs.
  • Phase 2: zone_boundaries migration + model: site_id, floor_plan_id, name, color, x, y, width, height. CRUD controller.
  • Phase 3: Canvas rendering: wrap existing FloorPlanView image + device dots. Add zone rectangles overlay from zone_boundaries data.
  • Phase 4: Zone drawing: click-drag to create rectangle. Resize handles. Select/deselect. Properties panel.
  • Phase 5: Device placement: drag from sidebar to canvas. Auto-zone assignment via hit testing (is the drop point inside a zone rectangle?).
  • Phase 6: Batch save: collect all position + zone changes, single POST to persist. Undo stack for last 20 actions.
  • Phase 7: Polish: zoom/pan, keyboard shortcuts, floor plan upload from canvas, empty states, confirmation dialogs for delete.
