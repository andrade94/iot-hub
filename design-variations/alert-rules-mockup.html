Design Notes

Key Changes from Current

Index: Cards → Table Table is denser, scannable, sortable. Cards waste vertical space with 5 rules. Table shows all info in one row: name, severity, conditions summary, scope, cooldown, toggle.
Index: Search + Filter Search by name + severity segmented filter. Same pattern as site show alerts.
Index: Row click → Show Click anywhere on the row to view the rule. Edit/delete buttons appear on hover (right column).
Create: Metric Combobox Metric input becomes a searchable combobox suggesting known metrics (temperature, humidity, co2, door_status, current, battery_pct, etc.) with units. Still allows freetext for custom metrics.
Create: Rule Preview Human-readable preview of what the rule will do: "When temperature is above 8 on all devices, trigger a critical alert. Cooldown: 30 min."
Show: Recent Alerts Shows the last 5 alerts triggered by this rule — with device name, value, time, status. Links to full alert detail. Answers "is this rule actually firing?"
Show: Alert count in sidebar Sidebar shows total alerts triggered count. Quick indicator of rule effectiveness.
