Design Notes
Device naming convention Name = asset tag (Astrea Temp-01). Label = position (Left wall, top shelf). Model = hardware (EM300-TH). Zone = location (Cooler A). Recipe = profile (Walk-in Cooler). All shown in the subtitle.
KPI strip with live readings Shows latest value for each metric the sensor measures + battery + signal. Large display numbers with units. Min/avg/max shown below for context. Battery has progress bar.
New fields: label + serial Label is freetext position descriptor within zone. Serial is physical hardware serial number. Both in the register form and detail sidebar.
Register form with all relationships Name + Label, Model (from sensor models catalog) + Serial, DevEUI + AppKey, Zone + Gateway + Recipe — all as selects from existing data.
Replace device workflow When hardware fails, admin enters new DevEUI + AppKey. All config transfers — name, zone, recipe, gateway, floor position, alert rules. Old device marked as replaced.
Detail sidebar links Recipe name links to recipe detail. Site name links to site. Gateway links to gateway show. Floor links to layout editor. Everything connected.
