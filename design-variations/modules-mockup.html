Design Notes
Card layout vs table Cards work better here than a table — each module is a "product" with icon, description, pricing. Cards give visual hierarchy and room for the toggle.
Active vs Available split Active modules at full opacity on top. Available modules at reduced opacity below. Modules with no matching devices at lowest opacity — can still be activated but won't do anything until devices are deployed.
Matching devices count Each card shows how many devices at this site would be affected. Green = has matching devices. Amber = has some devices ready. Red "No matching devices" = module would be inactive even if toggled on.
Activation preview Before activating, show exactly what will happen: alert rules created, devices affected, dashboard unlocked, billing impact. Same preview pattern as "Generate from Recipes".
Deactivation warning Red warning showing what will be removed: alert rules deleted, devices unmonitored, reports stop, billing reduction.
Reports link Active modules with dashboards show a "Reports →" link to their module dashboard (IAQ, Industrial). Modules without dashboards don't show this link.
