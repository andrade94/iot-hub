📐 Complete Review — Changes from v1 Mockup

  • KPI Strip fixed: Removed hardcoded zeros. Now shows: Devices (kpis.total_devices), Online % (kpis.online_count/total with x/y subtitle), Alerts (kpis.active_alerts), Low Battery (kpis.low_battery_count — was missing), Work Orders (myRequests.length), Gateways (site.gateways.length).
  • Floor plan selector: Added floor tabs (Planta Baja | Piso 2) for multi-floor sites. Each tab loads the corresponding floor plan image and its placed devices.
  • Unplaced device indicator: Header shows ⚠ 2 sin ubicar in amber when some devices have floor_x=null. Gives the user a reason to go place them.
  • Floor plan device filter bug: Controller must filter by floor_id, not just floor_x !== null. Each floor plan should only show its own devices.
  • Zone pills — observed range: Changed from fake "acceptable range" (2-5°C) to actual observed min X · max Y from ZoneMetricSummary. This is real data.
  • Zone pills — non-temperature zones: Compressor zones show 1.2 kW with ⚡ energia label. Office/Vitrina with no data show — with sin datos. The primary metric comes from the first item in zone.summary[].
  • Zone strip overflow: Uses overflow-x: auto with thin scrollbar + right fade edge. Works for 8+ zones without wrapping.
  • Alert limit transparency: Filter bar shows 5/7 count AND italic note mostrando 10 mas recientes so users know they're not seeing everything. "Ver todas las alertas" link at bottom navigates to full alert page.
  • Section divider: Says "PLANO DEL SITIO" when floor plans exist, falls back to "ZONAS" when they don't.
  • Zone cards (Variant B): Range now shows observed min · avg · max from real ZoneMetricSummary data. Non-temp zones show their actual metric + unit.
  • Empty site (Scenario 3): Shows centered empty state with CTA to manage devices.
  • All offline (Scenario 4): Full red health bar, 0% online. Demonstrates the critical state clearly.

✅ Data Sources for Implementation

  • kpis.total_devices, kpis.online_count, kpis.active_alerts, kpis.low_battery_count — from ChartDataService.getSiteKPIs()
  • site.gateways.length — already loaded via $site->load(['gateways'])
  • myRequests.length — already passed as prop
  • zone.summary[0] — primary metric per zone (first item in ZoneMetricSummary array)
  • zone.summary[0].min/max — observed range from readings (NOT compliance thresholds)
  • floorPlans[].devices — must be filtered by floor_id in controller
  • Unplaced count: allDevices.filter(d => d.floor_x === null).length
