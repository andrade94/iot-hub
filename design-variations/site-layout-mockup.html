📐 Site Layout Manager — Design Decisions

  • Full-page editor — Not a tab on the site show page. This is a dedicated tool accessed via Setup tab → "Floor Plans" config card, or the "Manage" button on the floor plan header.
  • Three-panel layout — Left sidebar (devices/zones/floors), center canvas (floor plan), right properties (selected zone/device details). Follows the Figma/design-tool mental model operators already know.
  • Drag to place — Unplaced devices in the sidebar have a dashed amber border. Drag them onto the canvas to place. The existing FloorPlanView already handles this — we wrap it in this layout.
  • Zone rectangles on canvas — Zones are drawn as rectangles (not polygons) over the floor plan image. Each zone has: name, color, position (x/y/w/h as percentages). Clicking a zone selects it in the right panel.
  • Auto-zone assignment — When a device is placed inside a zone rectangle, it auto-assigns device.zone to that zone's name. Moving a device to a different zone rectangle reassigns it.
  • Zone as data model — Currently zone is a string on Device. For zone boundaries, we need a new table: zone_boundaries with site_id, floor_plan_id, name, color, x, y, width, height. The zone string on Device stays for backward compatibility.
  • Rooms vs Zones — The mockup shows rooms with temperature overlays. These ARE the zones. Each zone boundary corresponds to a physical area on the floor plan.
  • Status bar — Bottom bar shows selected zone info + keyboard shortcuts. Contextual help.
  • Sidebar tabs — Three modes: Devices (drag to place, see unplaced vs placed), Zones (list with readings, click to select/highlight on canvas), Floors (manage floor plan images, upload, delete, reorder).
  • Save/Reset — Changes are batched locally. "Save Changes" persists all position + zone changes at once. "Reset" discards unsaved changes. This prevents accidental modifications.
  • Properties panel — Right side shows details of selected zone or device. For zones: name, color, devices in zone, boundary coordinates, delete. For devices: name, model, zone assignment, position, status.

✅ Implementation Plan

  • Phase 1: New route GET /sites/{site}/layout → dedicated Inertia page sites/layout.tsx
  • Phase 2: Create zone_boundaries migration + model (site_id, floor_plan_id, name, color, x, y, width, height)
  • Phase 3: Build the three-panel layout wrapping existing FloorPlanView
  • Phase 4: Add zone rectangle drawing (click-drag to create, resize handles)
  • Phase 5: Auto-zone assignment when device placed inside boundary
  • Phase 6: Batch save endpoint for positions + zone assignments
  • Phase 7: Update Overview tab floor plan to show zone overlays with readings
