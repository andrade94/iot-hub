# Internationalization (i18n)

This template includes full internationalization support using `laravel-react-i18n`, allowing you to build multi-language applications easily.

## Overview

- **Package**: `laravel-react-i18n`
- **Languages Included**: English, Spanish (French and German configured but need translation files)
- **Server-side Management**: Laravel handles locale detection and persistence
- **Client-side Integration**: React components automatically use translations

## Quick Start

### Using Translations in Components

```tsx
import { useLang } from '@/hooks/use-lang';

export default function MyComponent() {
    const { t } = useLang();

    return (
        <div>
            <h1>{t('Welcome')}</h1>
            <p>{t('Hello :name', { name: 'John' })}</p>
        </div>
    );
}
```

### Adding the Language Switcher

```tsx
import { LanguageSwitcher } from '@/components/ui/language-switcher';

export default function Header() {
    return (
        <header>
            <LanguageSwitcher showLanguageName />
        </header>
    );
}
```

## Configuration

### Available Languages

Edit `resources/js/components/ui/language-switcher.tsx` to add/remove languages:

```typescript
const languages = [
    { code: 'en', name: 'English', flag: '🇺🇸' },
    { code: 'es', name: 'Español', flag: '🇪🇸' },
    { code: 'fr', name: 'Français', flag: '🇫🇷' },
    { code: 'de', name: 'Deutsch', flag: '🇩🇪' },
] as const;
```

Also update the middleware `app/Http/Middleware/SetLocale.php`:

```php
protected array $availableLocales = ['en', 'es', 'fr', 'de'];
```

### Translation Files

Translation files are located in `lang/` directory:

```
lang/
├── en.json  (English)
├── es.json  (Spanish)
├── fr.json  (French - to be completed)
└── de.json  (German - to be completed)
```

### Adding New Translations

1. Add the key-value pair to all language files:

**lang/en.json:**
```json
{
    "My New Key": "My New Value"
}
```

**lang/es.json:**
```json
{
    "My New Key": "Mi Nuevo Valor"
}
```

2. Use in your components:

```tsx
const { t } = useLang();
t('My New Key');
```

## Usage

### useLang Hook

The `useLang()` hook provides a simple API for translations:

```tsx
import { useLang } from '@/hooks/use-lang';

function MyComponent() {
    const { t, locale, setLocale, has, tOrDefault } = useLang();

    // Simple translation
    const welcome = t('Welcome');

    // With replacements
    const greeting = t('Hello :name', { name: 'John' });

    // Get current locale
    const currentLang = locale(); // 'en'

    // Change locale programmatically (not recommended - use LanguageSwitcher)
    setLocale('es');

    // Check if translation exists
    if (has('Some Key')) {
        // Key exists
    }

    // Get translation or default
    const text = tOrDefault('Missing Key', 'Default Value');

    return <div>{welcome}</div>;
}
```

### LanguageSwitcher Component

```tsx
import { LanguageSwitcher } from '@/components/ui/language-switcher';

// Basic usage
<LanguageSwitcher />

// With language name
<LanguageSwitcher showLanguageName />

// With custom styling
<LanguageSwitcher
    variant="outline"
    size="sm"
    className="ml-auto"
/>
```

**Props:**
- `showLanguageName?: boolean` - Show language name next to icon
- `variant?: 'default' | 'outline' | 'ghost'` - Button variant
- `size?: 'default' | 'sm' | 'lg' | 'icon'` - Button size
- `className?: string` - Additional CSS classes

### Server-Side Locale Detection

The `SetLocale` middleware automatically detects the user's preferred language based on:

1. **Session** (user's explicit choice via LanguageSwitcher)
2. **User Model** (if user has a locale preference stored)
3. **Browser** (Accept-Language header)
4. **App Default** (config/app.php)

### Persisting User Preference

When a user changes language via the `LanguageSwitcher`, it:
1. Updates the session
2. Calls `POST /locale` to persist on server
3. Reloads the page to apply translations

## Backend Setup

### Middleware Registration

The `SetLocale` middleware needs to be registered. Add to `bootstrap/app.php`:

```php
->withMiddleware(function (Middleware $middleware) {
    $middleware->web(append: [
        \App\Http\Middleware\SetLocale::class,
    ]);
})
```

### Routes

Add the locale update route to `routes/web.php`:

```php
use App\Http\Controllers\LocaleController;

Route::post('/locale', [LocaleController::class, 'update'])->name('locale.update');
```

### Sharing Locale with Inertia

In `app/Http/Middleware/HandleInertiaRequests.php`, share the locale:

```php
public function share(Request $request): array
{
    return [
        ...parent::share($request),
        'locale' => app()->getLocale(),
    ];
}
```

## Advanced Usage

### Pluralization

Laravel's JSON translation files support pluralization using `|`:

**lang/en.json:**
```json
{
    "apple": "one apple|many apples"
}
```

Usage:
```tsx
const { t } = useLang();
t('apple', { count: 1 }); // "one apple"
t('apple', { count: 5 }); // "many apples"
```

### Nested Replacements

```tsx
t('Welcome to :app_name, :user_name!', {
    app_name: 'My App',
    user_name: 'John'
});
```

### Storing User Locale Preference

To store locale in User model, add a migration:

```php
Schema::table('users', function (Blueprint $table) {
    $table->string('locale', 5)->default('en');
});
```

Add methods to User model:

```php
public function getLocale(): string
{
    return $this->locale;
}

public function setLocale(string $locale): void
{
    $this->locale = $locale;
}
```

## Best Practices

1. **Use Translation Keys**: Always use translation keys, even in English
   ```tsx
   ❌ <h1>Welcome</h1>
   ✅ <h1>{t('Welcome')}</h1>
   ```

2. **Descriptive Keys**: Use descriptive keys for context
   ```json
   {
       "auth.login.title": "Login to your account",
       "auth.login.button": "Sign In"
   }
   ```

3. **Avoid Hardcoded Text**: Never hardcode user-facing text
   ```tsx
   ❌ <button>Click me</button>
   ✅ <button>{t('Click me')}</button>
   ```

4. **Test All Languages**: Regularly test your app in all supported languages

5. **Keep Files in Sync**: When adding keys, add to all language files

## Troubleshooting

### Translations Not Updating

1. Clear Vite cache:
   ```bash
   rm -rf node_modules/.vite
   npm run dev
   ```

2. Hard reload browser (Cmd/Ctrl + Shift + R)

### Missing Translations

If a translation key is not found, the key itself is returned:
```tsx
t('Missing Key') // Returns: "Missing Key"
```

Use `tOrDefault()` to provide fallbacks:
```tsx
tOrDefault('Missing Key', 'Fallback Text')
```

### Locale Not Persisting

Check that:
1. `SetLocale` middleware is registered
2. `/locale` route exists
3. Session is working properly

## Example: Multi-language Page

```tsx
export default function Example() {
    const { t } = useLang();

    return (
        <AppLayout title={t('Dashboard')}>
            <div className="p-6">
                <h1 className="text-2xl font-bold">{t('Welcome')}</h1>
                <p className="mt-4">{t('You\'re logged in!')}</p>

                <Button className="mt-4">
                    {t('Save')}
                </Button>
            </div>
        </AppLayout>
    );
}
```
