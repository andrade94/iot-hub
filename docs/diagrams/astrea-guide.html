System Guide

Architecture, modules, data pipeline, and role overview for the Astrea IoT Platform.

Platform Overview

Astrea is a multi-tenant LoRaWAN operations platform that bridges physical sensors and the people who make operational decisions. It serves retail chains, convenience stores, foodservice operations, industrial plants, commercial real estate, and pharmacies.

The platform follows four key principles:

📱

Three Interfaces

WhatsApp for emergencies and acknowledgment. Mobile app for daily monitoring. Web for configuration, analysis, and reports.

🔌

Modular by Segment

Cold Chain, Energy, Compliance, Industrial, IAQ, Safety, People -- activated per client organization.

🎯

Command Center

Astrea super admins see everything: all clients, all segments, all health metrics. Full operational oversight.

🔒

Strict Multi-Tenancy

Clients only see their own data. Scoping via org_id + site_id enforced by middleware.

Architecture

Data Pipeline

Sensors (LoRaWAN) --> Gateways (4G/Solar) --> ChirpStack Cloud (Managed) | | MQTT v Laravel MQTT Listener --> ProcessSensorReading job | +---> TimescaleDB (permanent time-series storage) +---> Redis HSET (hot cache, <1ms dashboard reads) +---> EvaluateAlertRules (rules engine) | +---> AlertRouter --> WhatsApp / Push / Reverb +---> SensorReadingReceived broadcast (live dashboards)

Storage Strategy

PostgreSQL + TimescaleDB

Single database for relational + time-series data. Hypertable with weekly chunks. Compression after 7 days (90%+ reduction). Continuous aggregates at 15m, 1h, and 1d intervals.

Redis Hot Cache

Latest readings cached per device with 15-minute TTL. Dashboard loads in <1ms. Reduces database reads by ~90%. Also handles queues, sessions, and alert rule state.

Auto-Resolution

ReadingQueryService picks the right data source: raw for 1h, 15m aggregate for 24h, 1h aggregate for 7-30d, and 1d aggregate for yearly views. All chart queries <100ms.

Real-Time

Laravel Reverb provides the WebSocket server. Three broadcast events push live data to frontends:

SensorReadingReceived

Live sensor values broadcast on site and device channels for real-time dashboard updates.

AlertTriggered

New alerts broadcast on site channels. Triggers desktop notifications and dashboard alert counts.

NotificationCreated

User-specific notifications for the bell icon. Delivered via private user channels.

Multi-Tenancy

Manual scoping (no package) using org_id and site_id columns. Two middleware enforce isolation:

EnsureOrganizationScope

Binds the current organization into the container. Regular users are scoped to their org_id. Super admins can switch organizations via session or header.

EnsureSiteAccess

Verifies the user can access the requested site. Super admins pass through. Org admins get all sites. Other roles checked against user_sites pivot table.

Roles Overview

7 roles across two ownership categories (Astrea internal and client organizations). See the Roles page for the full permission matrix.

super_admin
Full platform access. All orgs, all sites. Command Center.
support
Astrea support staff. Read access to all orgs for troubleshooting.
account_manager
Manages client relationships. Billing and org-level config.
technician
Field tech. Device diagnostics, work orders, battery/signal info.
client_org_admin
Client org admin. All sites in org, billing, users, config.
client_site_manager
Regional manager. Multiple assigned sites, dashboards, reports.
client_site_viewer
Store manager. Single site, read-only, acknowledge alerts.
