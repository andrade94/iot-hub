Frequently Asked Questions

Common questions about the Astrea IoT Platform -- from architecture to deployment to compliance.

General

What is Astrea IoT Hub? +

Astrea is a multi-tenant LoRaWAN operations platform that bridges physical sensors and the people who make operational decisions. It serves retail chains, convenience stores, foodservice operations, industrial plants, commercial real estate, and pharmacies.

The platform provides real-time monitoring, automated alerts, compliance reporting, and operational analytics through three interfaces: web application, mobile app, and WhatsApp.

How does Astrea work? +

LoRaWAN sensors (temperature, humidity, current, door contact, CO2, etc.) transmit readings to gateways every 5-15 minutes. Gateways relay data via 4G/Ethernet to ChirpStack Cloud, which forwards MQTT messages to the Laravel application.

The application decodes sensor data, stores it in PostgreSQL + TimescaleDB, caches the latest reading in Redis, and evaluates alert rules in real-time. When a threshold is breached, the alert router sends notifications via WhatsApp, push, email, and WebSocket.

What industries does Astrea support? +

Astrea supports any industry that needs environmental monitoring. Current segment modules include:

Cold Chain -- Retail, convenience stores, pharmacies, foodservice, cold storage warehouses.

Energy -- Equipment monitoring, compressor duty cycles, energy cost optimization.

Compliance -- COFEPRIS, NOM-251, HACCP, LFPDPPP regulatory compliance.

Industrial -- Manufacturing, vibration, compressed air, environmental monitoring.

IAQ -- Offices, schools, commercial real estate (CO2, PM2.5, humidity).

Safety & People -- Door monitoring, occupancy counting, traffic flow.

Sensors & Hardware

What sensors are supported? +

Astrea supports Milesight LoRaWAN sensors out of the box, decoded via the DecoderFactory pattern:

EM300-TH -- Temperature and humidity (cold chain, IAQ)

CT101 -- Current transformer for energy/compressor monitoring

WS301 -- Door/window contact sensor

AM307 -- Indoor air quality (CO2, PM2.5, TVOC, humidity, temperature)

VS121 -- People counting / traffic sensor

Additional sensor models can be added by implementing a new decoder class.

What gateways are used? +

Milesight UG65 (indoor, 4G/Ethernet backhaul) and SG50 (outdoor, solar-powered) gateways. Each site typically has one gateway included in the base fee, with additional gateways available at $2,500 MXN/month as add-ons.

Gateways connect to ChirpStack Cloud -- no self-hosted network server required.

Alerts & Monitoring

How fast are alerts delivered? +

Alerts are evaluated in real-time as each sensor reading arrives. From the moment a threshold breach is detected, WhatsApp and push notifications are typically delivered within 10-30 seconds. Web dashboard updates are instant via WebSocket (Laravel Reverb).

Duration-based thresholds require sustained breaches (e.g., temperature above -8C for 10 minutes) to avoid false alarms from momentary spikes.

Can I customize alert rules? +

Yes. Alert rules support three types: simple (threshold + duration), correlation (multi-sensor conditions), and baseline (learned patterns with anomaly detection at 2+ standard deviations).

Astrea provides pre-built alert recipes per sensor model and vertical. Organization admins can adjust thresholds, durations, and severity levels. They can also enable/disable individual rules per site via recipe overrides. Only super_admins can create entirely new recipes.

How does escalation work? +

Each site has an escalation chain with multiple levels. When an alert fires, Level 1 contacts are notified immediately. If the alert is not acknowledged within the configured delay (e.g., 15 minutes), it escalates to Level 2, and so on.

Each level can use different channels: WhatsApp, push notification, email, or SMS. WhatsApp messages include inline ACK/ESC keywords for one-tap response.

What about false alarms from defrost cycles? +

Commercial refrigerators defrost 2-4 times daily on fixed schedules, causing temperature spikes that look like failures. Without detection, each installation would generate 8-16 false alarms per day.

Astrea automatically learns defrost patterns within 48 hours of sensor installation. The system collects temperature spikes, compares Day 1 vs Day 2 patterns, and auto-generates defrost schedules when it detects consistent cycles. During confirmed defrost windows, cold-chain alerts are suppressed.

Security & Data

How is data secured? What about multi-tenancy? +

Astrea uses manual multi-tenancy scoping with org_id and site_id columns on all data tables. Two middleware enforce isolation:

EnsureOrganizationScope binds the current organization to every authenticated request.

EnsureSiteAccess verifies site-level permissions via the user_sites pivot table.

Session-based authentication (Laravel Fortify) for web, token-based (Laravel Sanctum) for mobile API. Two-factor authentication (TOTP) is supported. All passwords are hashed with bcrypt. Device app_keys are stored encrypted.

Is Astrea LFPDPPP compliant? +

Yes. The platform includes LFPDPPP (Mexican Federal Law for Protection of Personal Data) consent management. New users must accept the privacy policy before accessing the platform. Consent is timestamped and recorded. The privacy acceptance page (/privacy/accept) is enforced via middleware.

Activity logs track all data access and modifications with user, timestamp, IP address, and change details for audit purposes.

How long is sensor data retained? +

Raw sensor readings are retained for 12 months. After 7 days, data is compressed (90%+ reduction in storage). Continuous aggregates provide pre-computed rollups:

15-minute aggregates -- retained 3 months (for detailed 24h charts)

1-hour aggregates -- retained 5 years (for weekly/monthly charts)

1-day aggregates -- retained 10 years (for yearly KPI dashboards)

Reports & Compliance

What reports are available? +

Astrea generates branded PDF reports via Browsershot (Puppeteer), including:

HACCP temperature log -- Continuous temperature records with excursion flagging for NOM-251/COFEPRIS audits.

Energy reports -- Consumption analysis with peak/off-peak breakdown and night waste detection.

Alert analytics -- Trends, MTTR (mean time to resolve), top offending devices, and severity distribution.

Morning/regional/corporate summaries -- Automated daily digests delivered by email and push notification.

Reports can be generated on-demand or scheduled for automatic delivery (daily, weekly, monthly) via the Report Schedules settings page.

Mobile & Integration

Is there a mobile app? +

Yes. The Astrea companion app is built with Expo SDK 54 (React Native) and runs on iOS and Android. It connects to the platform via Sanctum API tokens with 26 endpoints.

Key features: real-time sensor readings, push notifications for alerts, work order management with photo capture, device status monitoring, and alert acknowledgment.

What integration options are available? +

Astrea supports several integration paths:

SAP export -- JSON export format compatible with SAP Business One for invoice synchronization.

CONTPAQi export -- Export format for CONTPAQi accounting software.

Data export -- CSV/JSON bulk export of sensor readings, alerts, and operational data.

Webhook subscriptions -- Event-type filtering for real-time integration (planned Phase 13).

Public REST API -- Versioned API with OpenAPI documentation (planned Phase 13).

All exports are non-blocking -- if an external system is unavailable, a local JSON copy is stored for retry.

Does Astrea support WhatsApp notifications? +

Yes. Critical and high-severity alerts are delivered via WhatsApp using the Twilio API (MVP) with migration to Meta Cloud API planned for scale. WhatsApp messages include alert details and inline ACK/ESC action keywords.

WhatsApp is also used for morning summaries and escalation notifications. Users can be set up as WhatsApp-only contacts (no app login required) for roles like night guards or external repair technicians.

Pricing & Deployment

What is the pricing model? +

Astrea uses a B2B subscription model with per-sensor pricing:

Base fee -- Per organization, varies by segment and contract type (monthly or annual).

Per-sensor fees -- EM300-TH: $150/mo, CT101: $200/mo, WS301: $100/mo, AM307: $175/mo, VS121: $125/mo (MXN).

Gateway add-ons -- $2,500/mo per additional gateway beyond the included one.

Invoicing is via CFDI (Mexican electronic invoice) with timbrado through Facturapi. Payment is via SPEI bank transfer -- no online payment processor.

How is the platform deployed? +

Production: Laravel Forge on DigitalOcean droplets with automated provisioning, SSL, deploys, and cron jobs. PostgreSQL 16 + TimescaleDB, Redis, and Laravel Reverb (WebSocket server).

Development: Laravel Herd provides local PHP 8.4, PostgreSQL, and Redis. Sites are served at http://[directory-name].test.

Monitoring: Laravel Telescope for debugging, Sentry for error tracking. File storage on S3-compatible DigitalOcean Spaces.

How does the platform scale? +

TimescaleDB compression (90%+ storage reduction), continuous aggregates (pre-computed rollups), and Redis hot caching keep database load manageable. The platform supports:

MVP (0-50 sites) -- Up to 750 sensors, 4 vCPU, 8GB RAM

Growth (50-200) -- Up to 3,000 sensors, 16GB RAM

Scale (200-500) -- Up to 7,500 sensors, 32GB RAM, daily chunks

Enterprise (500-1000) -- Up to 15,000 sensors, 64GB RAM, read replica

The bottleneck is server cost, not technology -- TimescaleDB handles billions of rows.
