Organization Setup

Create the client organization, configure branding, and set up billing -- Est. 1.5 hours

Create organization in Partner Portal

Navigate to Partner Portal as super_admin. Create new organization with name, slug, primary segment, default timezone, and default opening hour.

super_admin 10 min Partner Portal > New Organization

Configure organization branding

Upload client logo, set primary and secondary brand colors, choose font. This branding applies to all PDF reports and the client's web interface.

super_admin 10 min Settings > Organization

Create billing profile

Enter client's RFC, razon social, regimen fiscal, direccion fiscal, uso CFDI, and facturacion email. Multiple billing profiles supported if the client has multiple legal entities.

super_admin 15 min Settings > Billing > Profiles

Set up subscription

Configure base fee (by segment), discount percentage, and contract type (monthly or annual). Subscription auto-generates when the organization is created, but verify pricing.

super_admin 10 min Settings > Billing > Subscription

Create org_admin user account

Create the client's primary admin user (Director de Operaciones). Assign client_org_admin role. This user will manage their own sites, users, and configuration going forward.

super_admin 5 min Settings > Users > Create
2

Site Setup

Create sites, register gateways and devices, configure modules -- Est. 3 hours (varies by site count)

Create sites via onboarding wizard

For each physical location: enter site name, address, coordinates (lat/lng), timezone, opening hour, and optional segment override. The wizard guides through a 5-step process.

org_admin or super_admin 10 min/site Sites > New Site (Wizard Step 1)

Register gateways

For each site, register the LoRaWAN gateway: model (UG65/SG50), serial number, ChirpStack gateway ID. Mark as base (included) or add-on (billed at $2,500/mo). Verify the gateway appears as "online" in ChirpStack.

super_admin or technician 5 min/gateway Wizard Step 2 — Gateways

Register and provision devices

For each sensor: enter dev_eui, app_key (from sensor label), model, name, and zone. The system sends the app_key to ChirpStack for OTAA join. Wait for the first uplink to confirm connectivity.

super_admin or technician 3 min/device Wizard Step 3 — Devices

Upload floor plans (optional)

Upload site layout images per floor. Drag and drop sensors onto the floor plan to set their physical positions. Enables the temperature heatmap view.

org_admin or technician 10 min/site Wizard Step 3b — Floor Plans

Activate modules and apply recipes

Select which modules to activate for the site (Cold Chain, Energy, Compliance, etc.). Module activation auto-applies matching alert recipes to site devices based on sensor model.

org_admin 5 min/site Wizard Step 4 — Modules

Configure escalation chains

For each site, define the escalation chain: Level 1 contacts (immediate), Level 2 (after delay), Level 3 (final escalation). Set channels per level (WhatsApp, push, email).

org_admin 10 min/site Wizard Step 5 — Escalation

Use site templates for bulk setup

After setting up the first site, capture it as a template. Clone the template for remaining sites with similar configurations, then adjust individual settings as needed.

org_admin 2 min/site Settings > Site Templates
3

User Setup

Create user accounts, assign roles and sites, configure notification preferences -- Est. 2 hours

Create site manager accounts

Create regional manager users (client_site_manager). Each manages multiple assigned sites. Enter name, email, phone, and optional WhatsApp number.

org_admin 5 min/user Settings > Users > Create

Create site viewer accounts

Create store manager users (client_site_viewer). Each is assigned to exactly one site. These users do daily monitoring, alert acknowledgment, and maintenance requests.

org_admin 3 min/user Settings > Users > Create

Assign users to sites

Map each user to their sites via the user management page. Site managers get multiple sites, site viewers get one. Technicians get their service area sites.

org_admin 2 min/user Settings > Users > Edit > Sites

Add WhatsApp-only contacts

Create users with has_app_access=false for night guards, external repair technicians, or other contacts who only receive WhatsApp alerts. No password or app login needed.

org_admin 2 min/contact Settings > Users > Create (WhatsApp only)

Verify escalation chains reference correct users

Review each site's escalation chain to ensure the right users are at each level. Test by triggering a manual test alert and verifying delivery across all channels.

org_admin 5 min/site Settings > Escalation Chains
4

Go Live

Activate sites, verify monitoring, and confirm morning summaries are working -- Est. 1.5 hours

Verify all devices are reporting

Check the device list for each site. All devices should show "active" status with recent readings (within last 15 minutes). Investigate any "offline" or "pending" devices.

technician 10 min/site Devices page — filter by status

Wait for defrost pattern learning (48 hours)

Cold chain sites need 48 hours of data before defrost patterns are detected. After hour 49, the system auto-generates defrost schedules. Confirm with site viewers that detected patterns match reality.

system (automatic) 48 hours wait Settings > Defrost Schedules

Verify morning summaries are delivered

On the first full day after activation, confirm that morning summaries arrive at each site's opening hour. Check WhatsApp, push, and email delivery. Verify regional and corporate summaries also arrive.

org_admin + site_viewer 15 min Check WhatsApp + email + push
