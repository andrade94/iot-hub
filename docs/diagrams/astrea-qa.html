Phase Overview

Testing is organized into 8 phases, progressing from setup through authentication, configuration, monitoring, operations, and advanced features.

Phase 0: Setup Foundation
Environment preparation, database seeding, service startup, and credential verification.

Testing Approach

Per-Page Testing Template

For each page in the inventory, follow this structure:

1. How to get here: Direct URL + navigation path

2. Content (verify these exist): Checkbox list of expected UI elements

3. Actions and Destinations: Table of user actions, expected results, and navigation targets

4. Role differences: What each role sees (visibility matrix)

5. Business rules to verify: BR-xxx references with expected behavior

6. Screen states to test: Default, loading, empty, error, success, validation

7. Issues found: Space for documenting bugs and UX issues

Key Business Rules to Verify

The most critical rules that must be verified during QA:

BR-029: All routes scoped to user's organization (EnsureOrganizationScope)

BR-030: Site access validated via canAccessSite() -- super_admin=all, org_admin=org, others=pivot

BR-010: Alert rules evaluated per-reading, scoped to site + device

BR-011: Duration-based thresholds -- only sustained breaches trigger alerts

BR-013: Auto-resolution -- 2 consecutive normal readings resolve active alert

BR-017: Escalation chain levels with delay, users, and channels

BR-025: CFDI timbrado via Facturapi using billing profile

BR-047: Site onboarding is 5-step wizard (gateway, devices, floor plans, modules, escalation)

BR-048: Onboarding completion requires 1+ gateway, 1+ device, 1+ module
