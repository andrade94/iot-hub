Recommendation: Approach B

The full "Site Access" section in the Setup tab is the right choice for this platform. Here's why:

  • Primary use case is contextual — "Who has access to THIS site?" is always asked while looking at a site, not while browsing all users.
  • 3-15 users is the sweet spot — small enough to show inline without pagination, large enough to justify a dedicated section.
  • Role visibility matters — admins need to see the role breakdown at a glance (the compact version hides this).
  • Actionable in place — "Add User" button lets admins grant access without leaving the site page.
  • Setup tab has room — the existing config grid is 8 cards + site details. Adding a users section doesn't crowd it.
  • Also add the site filter to /settings/users as a secondary path — it's trivial and helps for cross-site user audits.
