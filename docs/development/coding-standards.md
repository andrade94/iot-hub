# Coding Standards

This document outlines the coding standards and conventions for this project.

## File Naming Conventions

### Directory Names

**All directories use lowercase** (industry standard for React/JS projects):

```
✅ resources/js/components/
✅ resources/js/pages/
✅ resources/js/layouts/
✅ resources/js/hooks/
✅ resources/js/utils/
✅ resources/js/types/
✅ resources/js/lib/
✅ resources/js/config/

❌ resources/js/Components/  # WRONG - use lowercase
❌ resources/js/Hooks/       # WRONG - use lowercase
```

**Note**: `resources/js/actions/`, `resources/js/routes/`, `resources/js/wayfinder/` are auto-generated by Laravel Wayfinder (gitignored)

### File Names

**React Components**: PascalCase (custom) or kebab-case (shadcn/ui)
```
✅ UserMenu.tsx        # Custom components
✅ DataTable.tsx
✅ button.tsx          # shadcn/ui components
✅ data-table.tsx

❌ userMenu.tsx        # WRONG for custom components
```

**Utility Files**: camelCase
```
✅ typeHelpers.ts
✅ errorHandling.ts

❌ TypeHelpers.ts
❌ error-handling.ts
```

**Custom Hooks**: kebab-case with 'use-' prefix
```
✅ use-toast.ts
✅ use-debounce.ts
✅ use-appearance.tsx

❌ useToast.ts         # WRONG - use kebab-case
❌ UseToast.ts
```

**Type Files**: camelCase
```
✅ pagination.ts
✅ navigation.ts

❌ Pagination.ts
```

## Import Conventions

### Always Use Lowercase Paths with @ Alias

```typescript
✅ import { Button } from '@/components/ui/button';
✅ import { useToast } from '@/hooks/use-toast';
✅ import type { User } from '@/types';
✅ import { cn } from '@/lib/utils';

❌ import { Button } from '@/Components/ui/button';  # WRONG - use lowercase
❌ import { useToast } from '@/Hooks/use-toast';     # WRONG - use lowercase
```

### Import Organization

Prettier automatically organizes imports via `prettier-plugin-organize-imports`:

```typescript
// 1. React and external libraries
import { useState, useEffect } from 'react';
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { z } from 'zod';

// 2. Internal imports (automatically sorted)
import { Button } from '@/components/ui/button';
import { useToast } from '@/hooks/use-toast';
import type { User } from '@/types';
import { cn } from '@/lib/utils';
```

Run `npm run format` to auto-organize imports.

## TypeScript Standards

### Strict Mode Enabled

This project uses strict TypeScript. Key rules:

- ✅ Always type function parameters and return values
- ✅ Use interfaces for object shapes
- ✅ Use type for unions and primitives
- ❌ Avoid `any` - use `unknown` and type guards instead
- ❌ Avoid type assertions unless absolutely necessary

### Component Props

Always define prop interfaces:

```typescript
// ✅ Good
interface ButtonProps {
    variant?: 'default' | 'outline' | 'ghost';
    size?: 'sm' | 'md' | 'lg';
    children: React.ReactNode;
    onClick?: () => void;
}

export function Button({ variant = 'default', size = 'md', children, onClick }: ButtonProps) {
    // ...
}

// ❌ Bad
export function Button(props: any) {
    // ...
}
```

### Type Imports

Use `type` keyword for type-only imports:

```typescript
✅ import type { User, Product } from '@/types';
✅ import type { Props } from './types';

❌ import { User, Product } from '@/types';  # If only using for types
```

## React Conventions

### Component Structure

```typescript
// 1. Imports
import { useState } from 'react';
import { Button } from '@/components/ui/button';
import type { User } from '@/types';

// 2. Type definitions
interface Props {
    user: User;
    onUpdate: (user: User) => void;
}

// 3. Component
export default function UserProfile({ user, onUpdate }: Props) {
    // 3a. Hooks
    const [isEditing, setIsEditing] = useState(false);
    const { toast } = useToast();

    // 3b. Event handlers
    const handleSave = () => {
        // ...
    };

    // 3c. Render
    return (
        <div>
            {/* JSX */}
        </div>
    );
}
```

### Hooks Usage

- ✅ Call hooks at the top level
- ✅ Use custom hooks for reusable logic
- ❌ Don't call hooks conditionally
- ❌ Don't call hooks in loops

```typescript
// ✅ Good
const { toast } = useToast();
const debouncedSearch = useDebounce(search, 300);

// ❌ Bad
if (condition) {
    const { toast } = useToast(); // WRONG
}
```

## Styling Conventions

### Tailwind CSS

Use Tailwind utility classes directly in JSX:

```tsx
// ✅ Good
<div className="flex items-center justify-between rounded-lg border p-4">
    <h2 className="text-lg font-semibold">Title</h2>
</div>

// ❌ Bad - avoid custom CSS unless absolutely necessary
<div className="custom-card">
    <h2 className="custom-title">Title</h2>
</div>
```

### Conditional Classes with cn()

Use the `cn()` utility for conditional classes:

```tsx
import { cn } from '@/lib/utils';

<button
    className={cn(
        'rounded-md px-4 py-2',  // Base classes
        variant === 'primary' && 'bg-blue-500 text-white',
        variant === 'secondary' && 'bg-gray-200 text-gray-800',
        disabled && 'opacity-50 cursor-not-allowed',
        className  // Allow override
    )}
>
    {children}
</button>
```

## Laravel (Backend) Conventions

### Controller Actions

```php
// ✅ Good - Return Inertia responses
public function index()
{
    return Inertia::render('Users/Index', [
        'users' => User::paginate(15),
    ]);
}

// ❌ Bad - Don't mix Blade views with Inertia
public function index()
{
    return view('users.index');
}
```

### Route Naming

```php
// ✅ Good - Named routes
Route::get('/users', [UserController::class, 'index'])->name('users.index');

// Frontend usage
<Link href={route('users.index')}>Users</Link>
```

### Validation

```php
// ✅ Good - Use Form Requests
$validated = $request->validate([
    'name' => 'required|string|max:255',
    'email' => 'required|email|unique:users',
]);
```

## Performance Best Practices

1. **Use React.memo for expensive components**
2. **Debounce search inputs** - Use `useDebounce` hook
3. **Lazy load routes** - Consider code splitting for large pages
4. **Optimize images** - Use appropriate formats and sizes
5. **Minimize re-renders** - Use proper dependency arrays in hooks

## Security Best Practices

1. **Never expose sensitive data** in frontend code
2. **Validate all inputs** on both frontend and backend
3. **Use CSRF protection** (Laravel handles this)
4. **Sanitize user input** before displaying
5. **Use parameterized queries** (Eloquent does this automatically)

## Accessibility Standards

1. **Use semantic HTML** elements
2. **Add ARIA labels** where needed
3. **Ensure keyboard navigation** works
4. **Maintain color contrast** ratios
5. **Test with screen readers** when possible
